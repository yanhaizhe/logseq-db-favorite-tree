# 技术方案

## 目标

实现一个面向 Logseq DB 图谱的收藏夹树插件：

- 根节点仅来自 Logseq 收藏夹页面
- 层级关系来自自定义页面属性，默认 `parent`
- 支持懒加载、当前页高亮、自动刷新、主题适配、循环检测、状态持久化

## 架构分层

### 1. Logseq 适配层

职责：

- 读取插件设置
- 读取收藏夹页面
- 获取当前页面
- 监听路由变化、主题变化、DB 变化
- 将 UI 状态持久化到插件 settings

### 2. 关系解析层

职责：

- 统一解析页面对象中的属性值
- 将 `parent` 属性归一化为父页面标题数组
- 构建 `parent -> children[]` 反向索引
- 兼容节点属性的多种返回结构（字符串、页面对象、数组）

### 3. 树状态层

职责：

- 维护根节点、展开状态、当前页、加载状态、错误状态
- 在首次展开时按需加载子节点
- 刷新时保留已展开节点并增量重建可见树
- 在递归构建时基于路径集合进行循环检测

### 4. 视图层

职责：

- 渲染右侧树面板
- 支持展开/折叠、点击跳转、刷新按钮
- 用左侧彩色边框高亮当前页
- 显示循环节点与空状态、加载状态

## 数据流

1. 插件启动
2. 注册 toolbar 按钮与设置项
3. 读取持久化状态（面板显隐、展开节点）
4. 读取收藏夹并生成根节点
5. 渲染树
6. 当用户展开某节点时：
   - 若该节点未加载过子节点，则扫描页面关系索引
   - 加载该节点的直接子页面
   - 若遇到祖先页面重复，则标记为循环并停止向下递归
7. 当 DB 变化、轮询触发或手动刷新时：
   - 重新抓取收藏夹与页面关系
   - 保留展开状态并刷新可见分支

## 懒加载策略

- 初始阶段只读取收藏夹根节点，不构建全树
- 第一次展开任意节点时，触发页面索引构建
- 后续展开直接命中内存索引
- 自动刷新会使索引失效，下一次需要子节点时重新构建

## 循环检测

- 递归展开时携带祖先路径集合
- 若待加入子节点标题已存在于当前路径，则：
  - 标记为 `cycle`
  - 在 UI 中展示“检测到循环”提示
  - 不继续递归加载该节点下层

## 持久化策略

通过 `logseq.updateSettings` 保存：

- `__panelVisible`
- `__expandedKeys`

这类内部状态不放入设置 schema，仅作为跨会话恢复数据。

## 兼容性取舍

- 优先支持 DB graph
- 优先使用官方收藏夹 API 与 DB 变化监听
- 页面属性值解析采用宽松归一化，避免因为 Logseq 小版本差异导致完全不可用
- 若实时事件缺失，5 秒轮询兜底
