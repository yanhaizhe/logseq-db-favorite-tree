# 技术方案

本文档描述 `Logseq DB Favorite Tree` 的目标、模块边界、关键数据流、状态模型、交互实现与后续演进方向，作为后续维护和迭代的基础设计说明。

## 1. 设计目标

### 1.1 核心目标

- 以 Logseq 收藏页作为树的根节点来源
- 以页面属性关系构建稳定的层级导航结构
- 提供比原生收藏夹更清晰的树形浏览能力
- 以悬浮面板 / 悬浮球方式常驻，不强依赖侧边栏
- 在不同 graph 下分别记忆使用状态，避免互相污染

### 1.2 非目标

- 不支持 file graph
- 不直接修改页面层级关系，只做读取和展示
- 不通过拖拽改写父子关系，拖拽仅用于同级排序
- 不承担通用文件树或数据库浏览器角色

## 2. 功能范围

- 收藏根节点加载
- 属性驱动层级关系解析
- 懒加载展开与循环检测
- 当前页高亮与多路径定位
- 树内搜索与祖先路径保留
- 默认标题排序与同级拖拽自定义排序
- 面板拖动、缩放、收回悬浮球、位置吸附
- 自动刷新、手动刷新、暂停自动刷新
- graph 级状态持久化
- 主题跟随与兼容性归一化

## 3. 总体架构

### 3.1 模块分层

#### 启动装配层

- `src/main.ts`
- 职责：注册设置、初始化插件实例、接线 DOM 事件、注册工具栏入口、处理插件生命周期

#### 业务协调层

- `src/plugin.ts`
- 职责：维护运行期状态、调度刷新、处理用户动作、协调树服务、布局服务与渲染层

#### 数据解析层

- `src/tree-service.ts`
- 职责：读取收藏根、解析页面属性、构建 `parent -> children` 索引、提供路径查找能力

#### 布局状态层

- `src/floating-layout.ts`
- 职责：管理面板与悬浮球的位置、尺寸、边界约束、吸附与拖拽缩放

#### 渲染层

- `src/render.ts`
- 职责：根据状态快照生成纯 HTML，保持 UI 与业务逻辑分离

#### 事件分发层

- `src/wire-dom-events.ts`
- 职责：从 DOM 事件中解析出具体交互语义，并转交给 `plugin.ts`

#### 设置与持久化层

- `src/settings.ts`
- 职责：读取用户设置、维护 graph 级内部状态、向 `logseq.updateSettings` 写回数据

#### 基础支撑层

- `src/constants.ts` / `src/types.ts` / `src/utils.ts` / `src/theme.ts` / `src/toolbar.ts`
- 职责：常量、类型、通用工具、主题同步与工具栏入口注册

### 3.2 模块依赖方向

- `main` 依赖 `plugin`、`wire-dom-events`、`toolbar`
- `plugin` 依赖 `settings`、`tree-service`、`floating-layout`、`render`
- `render` 只消费状态快照，不直接依赖 Logseq API
- `tree-service` 聚焦数据读取与关系解析，不承担 UI 状态管理
- `settings` 聚焦配置与持久化，不承担业务编排

该依赖方向保证了渲染层和数据层相对纯净，便于后续继续扩展功能而不让 `plugin.ts` 之外的模块承担过多协调逻辑。

## 4. 核心数据模型

### 4.1 配置项

当前公开设置项包括：

- `hierarchyProperty`：层级属性名，默认 `parent`
- `panelWidth`：悬浮面板默认宽度
- `pollIntervalSeconds`：自动刷新轮询间隔
- `sidebarPosition`：首次显示时默认靠左或靠右

### 4.2 运行期状态

运行期核心状态主要由 `plugin.ts` 维护，包括：

- 根节点列表
- 当前页标题
- 当前页路径
- 已展开节点集合
- 已加载节点集合
- 加载状态与错误状态
- 搜索关键字
- 最近定位节点
- 定位闪烁节点
- 自动刷新开关
- 功能区收起状态
- 排序映射
- 面板 / 悬浮球布局状态

### 4.3 graph 级持久化状态

graph 维度持久化数据包括：

- 展开状态
- 滚动位置
- 最近定位节点
- 视图模式
- 功能区收起状态
- 排序映射
- 面板位置与悬浮球位置
- 面板尺寸

其中图谱级状态通过 `__graphStates` 统一序列化保存，以避免不同 graph 相互覆盖。

## 5. 关键流程设计

### 5.1 启动流程

1. `main.ts` 注册设置 schema
2. 初始化插件实例
3. 绑定 DOM 事件
4. 注册工具栏按钮
5. 读取当前 graph 标识
6. 从设置中恢复 graph 级状态
7. 读取收藏页并生成根节点
8. 根据恢复状态渲染面板或悬浮球

### 5.2 树加载流程

1. 启动时先加载收藏根节点
2. 根节点按标题排序
3. 用户首次展开某节点时，确保子节点索引存在
4. 若索引尚未建立，则批量扫描页面属性并建立 `parent -> children[]`
5. 子节点按标题排序后返回
6. 若当前层存在自定义排序，则在默认排序基础上应用覆盖

### 5.3 当前页定位流程

1. 获取当前页面
2. 调用路径查找能力寻找所有命中路径
3. 将所有命中路径上的祖先节点标记为已展开
4. 记录当前定位节点
5. 触发渲染并滚动到目标节点
6. 若用户切换到其他页面，则自动清理旧定位标记

### 5.4 搜索流程

1. 输入搜索关键字
2. 进入搜索模式
3. 只保留命中节点及其祖先路径
4. 暂停拖拽排序入口
5. 渲染精简后的可见树
6. 清空关键字后恢复正常树浏览

### 5.5 排序流程

1. 默认按页面标题排序
2. 用户通过拖拽把手在同级内调整顺序
3. 将新顺序写入 `sortOrders`
4. graph 级持久化排序映射
5. 下次刷新时先走默认标题排序，再应用自定义顺序覆盖

### 5.6 布局交互流程

1. 面板支持拖动和右下角缩放
2. 标题栏双击可收回为悬浮球
3. 悬浮球点击恢复面板
4. 悬浮球拖动后自动吸附左右边缘
5. 所有布局结果按 graph 维度持久化

## 6. 排序方案设计

### 6.1 设计原则

- 默认排序必须稳定且可预测
- 自定义排序不能破坏新节点的自然接入顺序
- 同级排序与跨层级结构变更应明确区分
- 排序状态必须可以独立保存到 graph 维度

### 6.2 当前方案

- 根节点与子节点先统一走标题排序
- 使用 `ROOT_SORT_KEY` 代表根层排序
- 使用父节点的规范化 key 代表某一层级的排序空间
- `SortOrderMap` 只保存用户显式拖拽过的层级顺序
- 应用排序时先按自定义顺序输出已记录项，再追加未记录新项

### 6.3 方案收益

- 新增节点不会因为旧排序映射而消失
- 默认顺序和自定义顺序职责清晰
- 不需要修改底层 DB 数据
- 刷新后排序结果仍保持稳定

## 7. 搜索与定位方案

### 7.1 搜索策略

- 搜索基于页面标题匹配
- 搜索结果保留祖先路径，避免命中项脱离上下文
- 搜索模式与排序模式隔离，防止用户在筛选态误改真实顺序

### 7.2 定位策略

- 当前页定位使用全路径查找，而不是首条路径命中即停止
- 多命中路径全部展开，保证用户看到完整上下文
- 定位标签跟随当前页变化自动失效，避免误导

## 8. 刷新与一致性策略

### 8.1 刷新来源

- Logseq DB 变化事件
- 轮询定时器
- 用户手动刷新

### 8.2 一致性原则

- 刷新时尽量保留展开状态
- 刷新时重新计算收藏根和属性索引
- UI 状态与数据状态尽量解耦，避免因局部失败导致整个插件不可用

### 8.3 自动刷新控制

- 用户可在面板内暂停自动刷新
- 暂停状态会持久化
- 轮询间隔由设置项控制

## 9. 持久化方案

### 9.1 存储方式

- 使用 `logseq.updateSettings(...)`
- 公开设置项与内部状态分离
- 内部状态不暴露在设置面板 schema 中

### 9.2 持久化粒度

- 全局粒度：面板可见性、自动刷新暂停
- graph 粒度：布局、展开状态、排序、功能区收起、最近定位节点、滚动位置

### 9.3 兼容性处理

- 支持从旧版分散字段迁移到新的 graph 级状态结构
- 对非法或缺失字段进行默认值回填
- 对排序映射与面板尺寸做归一化修正

## 10. UI 与交互设计原则

- 面板交互要轻量，不遮挡主要写作流
- 常用动作放在顶部工具区
- 标题栏承担拖拽与收回入口，但对 `i` 提示图标等交互做隔离
- 树节点信息层次清晰，状态尽量通过轻量样式表达
- 搜索、定位、排序三类能力互不打架

## 11. 风险与约束

### 11.1 数据约束

- 层级关系完全依赖页面属性质量
- 收藏夹本身的返回结构可能随 Logseq 版本存在差异

### 11.2 交互约束

- 当前拖拽排序仅在同级内生效
- 当前没有从插件内直接维护收藏夹的能力
- 当前没有恢复默认排序入口

### 11.3 性能关注点

- 收藏根数量较多时的首屏渲染
- 深层路径定位时的批量展开成本
- 搜索和刷新同时发生时的状态一致性

## 12. 后续演进建议

详细优先级、范围边界与排期建议见 [产品路线图 PRD](./product-roadmap-prd.md)。

优先级较高的后续迭代建议如下：

- 增加“恢复默认排序”与“清除当前层排序”
- 增加从当前页一键加入收藏 / 从树内移出收藏
- 增加多命中路径切换导航
- 增加更明确的拖拽反馈与异常提示
- 针对大树场景补强性能与状态恢复兜底
