# 产品路线图 PRD

本文档定义 `Logseq DB Favorite Tree` 的产品方向、迭代优先级与排期建议，供开发排期与需求评审使用。

## 1. 产品定位

### 1.1 一句话定位

以 Logseq 收藏页为根、页面属性为边的**属性驱动层级导航树**。

### 1.2 核心价值

| 维度 | 描述 |
|---|---|
| 导航效率 | 将扁平收藏升级为可展开、可搜索、可排序的层级目录 |
| 零维护 | 层级关系来自页面属性，无需手工维护目录结构 |
| 低打扰 | 原生侧边栏常驻 + 悬浮面板/球可选，不影响写作流 |
| 多图隔离 | 不同 graph 独立记忆布局、排序与展开状态 |

### 1.3 非目标

- 不替代 Logseq 原生文件树或数据库浏览器
- 不修改页面原始层级数据
- 不支持 file graph
- 拖拽仅用于同级排序，不改变父子关系

## 2. 当前版本功能矩阵（v1.1.x）

详见 [功能清单](./feature-list.md)，核心能力包括：

- 收藏根 + 属性层级解析
- 侧边栏 / 悬浮面板 / 悬浮球三种形态
- 树内搜索 + 多命中导航 + 祖先路径保留
- 当前页定位 + 聚焦路径 + 折叠其他分支
- 默认标题排序 + 同级拖拽自定义排序 + 排序模式切换 + 清除自定义排序
- 节点下快捷新建子页面（含重名拦截与回滚）
- 多语言跟随 + graph 级状态持久化

## 3. 迭代优先级框架

| 优先级 | 定义 | 当前状态 |
|---|---|---|
| P0 | 影响排序/定位核心体验的补充能力 | 排序模式切换、清除排序已完成；落点反馈待补 |
| P1 | 提升日常使用频率的便利性 | 收藏管理入口待补 |
| P2 | 增强复杂场景体验的优化 | 大树性能、异常兜底待补 |

## 4. 排序治理专项（已落地）

### 4.1 设计原则

- 默认排序（标题序）始终作为基线，不可移除
- 自定义排序覆盖在默认排序之上，新增节点不受缺失映射影响
- 「切换到默认排序」保留已保存的自定义顺序，支持随时切回
- 「清除自定义排序」属于破坏性操作，需要二次确认
- 排序模式（默认/自定义）按层级独立保存，按 graph 隔离

### 4.2 当前实现对照

| 需求 | 状态 |
|---|---|
| 同级拖拽自定义排序 | 已实现 |
| 排序模式切换（默认 ↔ 自定义） | 已实现 |
| 清除自定义排序（含二次确认） | 已实现 |
| 当前层排序模式视觉状态标记 | 已实现 |
| 排序按 graph 独立持久化 | 已实现 |
| 拖拽落点反馈增强 | 待实现（P0） |

### 4.3 待补充

- 拖拽过程中更明显的落点位置提示（如高亮插入线、阴影目标行）
- 拖拽成功/失败后的短暂动画反馈

## 5. P1 迭代：收藏管理入口

当前插件不提供从面板内直接添加/移除收藏的能力，用户需要回到 Logseq 原生收藏界面操作。

### 5.1 建议方案

| 功能 | 入口 | 风险 |
|---|---|---|
| 从当前页一键加入收藏 | 定位按钮旁 / 当前页节点菜单 | Logseq API 兼容性 |
| 从节点快捷移出收藏 | 节点菜单 / 长按 | 误操作需确认 |
| 收藏状态视觉标识 | 节点图标 | 需额外 API 查询 |

## 6. P2 迭代：复杂场景增强

### 6.1 大树场景性能

- 根节点 > 50 时的首屏渲染优化
- 深层路径定位的批量展开开销控制
- 搜索索引增量更新而非全量重建

### 6.2 异常与空状态

- 更细粒度的加载失败原因提示（API 超时 / 权限 / 数据格式异常）
- 空搜索结果与空收藏状态的差异化引导

### 6.3 搜索体验增强

- 搜索结果命中路径高亮
- 搜索历史记录
