# 使用指南

本文档面向插件使用者，说明 `Logseq DB Favorite Tree` 的安装方式、基础概念、常用操作、设置项含义与常见问题。

## 1. 适用范围

- 适用于 `Logseq DB graph`
- 不适用于 `file graph`
- 页面层级关系来自页面属性，默认使用 `parent`

## 2. 安装方式

### 2.1 本地开发加载

1. 在项目目录执行安装依赖：

```bash
npm install
```

2. 执行构建：

```bash
npm run build
```

3. 在 Logseq 中开启开发者模式
4. 选择 `Load unpacked plugin`
5. 选择当前项目目录完成加载

## 3. 基础概念

### 3.1 根节点

- 插件会把 Logseq 收藏页作为树的根节点
- 只有被收藏的页面才会出现在第一层

### 3.2 子节点

- 子节点来自页面属性关系
- 默认读取 `parent` 属性
- 如果某页面声明其父页面为某个收藏页或某个树中页面，它就会作为对应子节点出现

### 3.3 当前页定位

- 当前打开的页面如果存在于收藏树中，可以通过“定位当前页”快速展开其路径
- 若同一页面通过不同路径被引用，插件会展开所有相关路径

## 4. 从零设置 `parent` 属性

这一节面向第一次使用属性组织页面层级的用户，目标是让一张页面出现在另一张页面下面。

### 4.1 先理解目标关系

- 假设你希望 `子页面A` 出现在 `父页面B` 下面
- 那么需要在 `子页面A` 上设置一个属性，属性名为 `parent`
- 这个属性的值填写为 `父页面B`
- 插件读取后，就会把 `子页面A` 挂到 `父页面B` 下面显示

可以理解为：

```text
子页面A
parent -> 父页面B
```

### 4.2 第一步：先准备父页面

1. 新建或打开你要作为上级目录的页面，例如 `项目管理`
2. 将这个页面加入 Logseq 收藏
3. 这样它才会作为收藏树的根节点出现

说明：

- 如果父页面没有加入收藏，它仍然可以作为某些页面的父节点
- 但它自己不会自动成为第一层根节点

### 4.3 第二步：在子页面中新增属性

1. 打开你要放到树里的子页面，例如 `周计划`
2. 在页面属性区域找到“添加属性”入口
3. 输入属性名：`parent`
4. 确认创建该属性

属性名要求：

- 默认必须叫 `parent`
- 如果你在插件设置里把“层级属性名”改成了别的名称，例如 `belongs-to`，那么这里也要使用同名属性
- 插件中的属性名和页面里实际填写的属性名必须一致

### 4.4 第三步：设置属性类型

推荐做法：

- 优先把 `parent` 设置成“页面引用”或 `Node` 类型
- 这样在属性值里直接选择页面，稳定性最好

具体建议：

- 如果 Logseq 属性编辑器里能选类型，优先选“引用页面对应的类型”
- 如果你的界面显示为 `Node`，就选 `Node`
- 如果你的界面显示为“页面引用”或类似名称，也优先选这一类

兼容性说明：

- 插件会优先适配 `Node` 类型
- 也兼容页面引用对象、数组值以及 `[[页面名]]` 这种引用文本
- 如果你的 Logseq 版本没有明显的类型选择入口，直接把值写成页面引用通常也可以识别

### 4.5 第四步：设置单值父页面

如果一个页面只有一个上级，按下面方式设置即可：

1. 在子页面的 `parent` 属性值中选择一个页面
2. 例如选择 `项目管理`
3. 保存后，`周计划` 就会显示在 `项目管理` 下

示例：

```text
页面：周计划
属性名：parent
属性值：[[项目管理]]
```

这适合最常见的一对一层级关系。

### 4.6 第五步：设置多值父页面

如果一个页面希望同时挂到多个父页面下面，可以把 `parent` 设置为多值。

适用场景：

- 一个页面需要同时出现在多个主题树下
- 例如 `会议纪要` 同时属于 `项目管理` 和 `团队协作`

设置步骤：

1. 打开目标子页面，例如 `会议纪要`
2. 找到 `parent` 属性
3. 将该属性改为允许多个值
4. 依次添加多个父页面
5. 例如添加 `项目管理` 和 `团队协作`

示例：

```text
页面：会议纪要
属性名：parent
属性值1：[[项目管理]]
属性值2：[[团队协作]]
```

设置完成后：

- 插件会把 `会议纪要` 同时挂到多个父页面下面
- 使用“定位当前页”时，也会展开所有相关路径

### 4.7 如果界面里看不到“多值”选项

不同版本的 Logseq 界面可能不完全一样，可以按下面原则处理：

- 如果属性支持连续添加多个页面值，就继续向同一个属性里添加多个页面
- 如果属性编辑器支持数组或多选，开启后添加多个页面
- 如果你的版本允许直接写引用文本，也可以写成多个页面引用

常见可识别写法包括：

```text
parent:: [[项目管理]]
parent:: [[项目管理]], [[团队协作]]
parent:: [[项目管理]]
         [[团队协作]]
```

说明：

- 插件会按多值页面引用来解析
- 如果同一个页面配置了多个父页面，它就会出现在多条路径中

### 4.8 第六步：在插件中验证结果

1. 确保父页面已加入收藏
2. 打开插件面板
3. 展开父页面
4. 检查子页面是否已出现在对应位置
5. 如未立即出现，可点击手动刷新

### 4.9 推荐配置方式

- 属性名统一使用 `parent`
- 属性值尽量使用页面引用，而不是纯文本
- 单父页面用单值
- 多父页面再开启多值
- 同一张图谱尽量统一写法，不要一部分页面用纯文本、一部分页面用不规范缩写

## 5. 快速上手

### 5.1 打开插件

- 在 Logseq 顶部工具栏点击插件图标
- 插件会以悬浮面板形式显示

### 5.2 浏览树结构

- 点击节点左侧展开按钮查看下级内容
- 点击页面标题可直接打开对应页面
- 当前打开页面会在树中高亮显示

### 5.3 定位当前页

- 点击工具栏中的定位按钮
- 插件会自动展开当前页所在的所有路径
- 目标节点会出现定位标记并短暂高亮

### 5.4 搜索页面

- 在搜索框输入页面标题关键字
- 插件仅展示命中页面及其祖先路径
- 清空搜索框后恢复完整树视图

## 6. 面板操作

### 6.1 拖动面板

- 按住标题栏拖动可改变面板位置
- 拖动结果会被记住

### 6.2 调整面板大小

- 拖动右下角可调整宽高
- 点击“默认尺寸”可恢复默认大小

### 6.3 收回悬浮球

- 点击面板收回按钮，或双击标题栏
- 面板会收回为一个悬浮球
- 点击悬浮球可再次展开面板

### 6.4 悬浮球拖动

- 悬浮球可自由拖动
- 拖动结束后会自动吸附到左侧或右侧边缘

## 7. 工具区说明

### 7.1 功能区收起

- 顶部功能区支持一键收起 / 展开
- 收起后可为树内容腾出更多空间

### 7.2 说明提示

- 标题栏左上角的 `i` 图标用于显示说明
- 鼠标停留或点击后可查看当前提示信息

### 7.3 刷新控制

- 支持手动刷新树内容
- 支持暂停 / 恢复自动刷新
- 自动刷新状态会被记住

## 8. 排序说明

### 8.1 默认排序

- 根节点默认按页面标题排序
- 子节点默认按页面标题排序

### 8.2 自定义排序

- 可以拖动节点左侧的排序把手
- 仅支持同级节点之间重排
- 自定义排序会覆盖当前层的默认顺序
- 排序结果会按 graph 独立记住

### 8.3 搜索中的限制

- 搜索状态下不能拖拽排序
- 这是为了避免筛选结果与真实树顺序混淆

## 9. 设置项说明

### 9.1 层级属性名

- 默认值为 `parent`
- 用于声明“当前页面的父页面是谁”
- 如果你的图谱使用其他属性名来组织层级，可在这里改成对应字段

### 9.2 面板宽度

- 控制面板默认宽度
- 用户手动缩放后的尺寸会另外记住

### 9.3 自动刷新间隔（秒）

- 控制轮询自动刷新的时间间隔
- 数值越小，树刷新越及时，但轮询频率也越高

### 9.4 初始侧向偏好

- 控制首次显示时默认靠左还是靠右
- 如果你已经拖动过面板或悬浮球，插件会优先使用最近记住的位置

## 10. graph 级状态记忆

插件会为不同 graph 分别记住以下状态：

- 面板位置
- 面板尺寸
- 悬浮球位置
- 展开状态
- 功能区收起状态
- 排序结果
- 视图模式
- 最近定位节点

这意味着你切换到另一个 graph 时，会看到该 graph 自己的树状态，而不是共用上一张图谱的布局和排序。

## 11. 常见问题

### 11.1 为什么页面没有出现在树里？

常见原因包括：

- 页面没有被加入收藏，因此不会成为根节点
- 页面虽然存在，但没有通过层级属性挂到某个已显示节点上
- 当前使用的层级属性名不是 `parent`，需要去插件设置中修改

### 11.2 为什么定位不到当前页？

常见原因包括：

- 当前页面不在收藏树覆盖范围内
- 当前页面没有通过层级关系连接到任何收藏根节点
- 页面标题与属性引用不一致，导致关系未被正确识别

### 11.3 为什么搜索时不能拖拽排序？

- 搜索结果只展示部分路径，并不是完整同级结构
- 为避免误操作，插件在搜索态禁用了拖拽排序

### 11.4 为什么切换 graph 后状态变了？

- 这是预期行为
- 插件会按 graph 分别记住布局、排序与展开状态

### 11.5 `parent` 应该填纯文本还是页面引用？

- 推荐填写页面引用
- 最稳妥的方式是选择页面引用对象或 `[[页面名]]`
- 纯文本在部分场景也可能被识别，但不如页面引用稳定

### 11.6 一个页面可以设置多个 `parent` 吗？

- 可以
- 只要你的属性支持多值，并且每个值都是有效页面引用，插件就会把它识别为多个父页面

## 12. 推荐使用习惯

- 用收藏夹维护导航入口，用属性维护层级关系
- 将常用主题页加入收藏，作为整棵树的一级入口
- 深层页面尽量保持统一的父属性写法
- 当树结构变化较频繁时，保留自动刷新开启
- 当你需要更大阅读区时，先收起功能区或收回悬浮球
